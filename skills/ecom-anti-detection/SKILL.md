# ecom-anti-detection — 电商防风控 playbook

## 核心原则

> **风控系统不是在检测"爬虫工具"，而是在检测"不像真人的行为"。**
> 每个动作前先自问：**真人会这样操作吗？** 不会，就是风控信号。
>
> 换个问法：不要问"怎么最快完成任务"，要问"一个真人为了这个目的会怎么逛"——然后照那个人的样子操作。

**状态卫生：** 你打开的每个弹窗 / 浮层 / 抽屉都是一个**未关闭的状态**。在做与它无关的下一步前，先关闭或完成它——真人的视野里不会同时存在"开着的弹窗"和"去点它后面的按钮"。

## 何时用

只要是大模型驱动浏览器自动化访问京东 / 淘宝 / 天猫——浏览、进商品详情、提评价、dogfood、评测、任意自动化访问——就加载本 skill。**工具无关**：把正文的中立动作映射到你手头工具的对应能力即可（vortex / playwright / 等）。不限于评测工作流。

## 何时不用

- 非这三个站点的普通网页自动化
- 需要破解滑块 / 验证码（本 skill 命中即升级人工，不破解）
- 想做指纹伪造 / UA 池 / 代理轮换（与"像真人"原则相悖，本 skill 不做）

## 强制前置自检回路（脊柱 — 每次访问目标站点前必走）

不是静态清单，而是把"像不像人"的判断嵌进每一次访问的执行路径：

```
1. 想清这次要做什么访问（导航 / 点击 / 翻页 / 提取…）
2. 自检：这样访问像不像一个真人在逛？
   ├─ 当前有没有未关闭的弹窗/浮层？有 → 先关闭或完成它，再继续
   ├─ 对照「反人类行为对照表」看有没有踩红旗
   └─ 对照「真人会怎么逛」推演看是不是直奔目标的最短路径
3. 不像人 → 先修正方案：
   换搜索页入口 / 换真实鼠标点击 / 加随机停顿 / 降频 / 调整会话弧线
4. 像人了 → 才真正调用工具执行
5. 执行后监测网络/控制台的风控信号 → 命中走「退避决策树」
```

**三层闭环：** 主动推演生成像人的方案 → 被动闸门执行前拦红旗 → 前置自检回路把两者强制串进每次访问，不像就修正再动手。

## 反人类行为对照表（被动闸门）

每行 = 一个"红旗 → 替代动作"。执行任何访问前对照检查，更要据此泛化到表外的新动作。

| 反人类行为（风控元凶） | 为什么是红旗 | 真人替代动作 |
|---|---|---|
| 直接导航/跳转到 `item.jd.com/xxx` / `item.taobao.com/item.htm?id=xxx` 商品详情 URL | 真人从不背商品 URL、冷跳进详情；只有脚本会 | 从搜索/列表页用真实鼠标点击进入，走站内 SPA 跳转 |
| 导航后立即连发动作，零停顿 | 真人要看、要读、要想；毫秒级连击只有机器 | 每步等页面稳定（idle）+ 随机化停顿 |
| 固定时间间隔的规律性操作 | 真人节奏天然抖动；钟表般规律是脚本指纹 | 停顿时长随机化，动作顺序带人类犹豫 |
| 合成 click 事件（非 `isTrusted`） | 风控读 `event.isTrusted`，合成事件一眼假 | 用能触发真实鼠标事件的方式点击 |
| 高频访问同一域、机械翻页 | 真人浏览量有限、会分心、会跳出 | 单会话自我限量，降频，必要时切平台 |
| 频繁清 cookie / 换 UA / 换指纹 | 真人指纹长期稳定；频繁变更=刻意规避 | 复用稳定登录 profile，不动指纹 |
| 弹窗/浮层未关就去操作页面其他区域 | 真人视野被弹窗占据、物理上也点不到被遮挡的元素；穿透点击只有脚本会 | 先关弹窗（Esc / 关闭按钮 / 点遮罩）回到主页面，再做下一步 |
| 风控拦截后立即硬重试 | 真人遇阻会停、会换路；机器才无脑重撞 | 命中 403/滑块即停，走退避决策树 |

## 主动人化推演（动手前先想"真人会怎么逛"）

对照表是被动闸门，但模型天然倾向走最短路径直奔目标，而"一步到位的最优路径"本身就是反人类指纹。所以动手前先在 thinking 里推演一个真人的浏览会话：

- **设人设与意图**：我是个想买 X 的真人，预算多少、为什么逛、在意什么 → 行为有动机锚点
- **规划真人节奏的会话弧线**：先搜 → 扫几个结果 → 点开 1–2 个看看 → 可能返回再比 → 可能被相关推荐带偏 → 才决定。**不是**搜完直接跳目标商品
- **保留犹豫与非线性**：真人会回退、改搜索词、停下读评价；机器的"最优直达"是指纹
- **显式写出来**：先在 thinking 说清"这个真人接下来会怎么动"，再落到具体工具调用

## 四层归纳（对照表的分类，便于记忆）

- **L1 人化路径与真实交互** — 入口走搜索页、真实鼠标点击、站内 SPA 跳转
- **L2 节奏 / 频控节流** — 等页面 idle + 随机停顿 + 单会话限量
- **L3 会话 / 登录态复用** — 复用稳定登录 profile、开场检测登录态、不动指纹
- **L4 风控检测 + 降级恢复** — 监测网络/控制台信号、命中即停、退避决策树

## 进站开场范例（前置自检回路的一个应用）

```
1. 登录态检测（L3）→ 读用户名节点 / 查 cookie；未登录则升级人工
2. 从搜索页进入（L1）→ 导航到 s.taobao.com/search?q=<品类> 或 search.jd.com
3. 等页面 idle（L2）等页面真稳定
4. 发现商品卡 → 真实鼠标点击进详情（L1）
5. 每步后监测 403/滑块/验证码信号（L4）→ 命中走退避决策树
6. 详情页数据做文本/评价提取，动作间随机化停顿（L2）
```

这是回路的一个应用示例，不是唯一正确流程。以"像不像真人"为准绳灵活变通。

## 工具无关说明

本 skill 不绑定任何具体自动化工具。把正文的中立动作映射到你手头工具的对应能力即可：

> 导航、真实鼠标点击、发现可交互元素、点击、文本/评价提取、等页面稳定、监测网络信号、查登录态/cookie
> —— 例如 vortex 的 observe/act/mouse_drag/extract/wait_for/debug_read，playwright 的 locator/click/waitForLoadState 等。

## 风控信号识别（用工具的网络/控制台监测能力捕获）

| 信号 | 识别签名 |
|---|---|
| 403 频控 | 网络请求出现 `pc-frequent-pro.pf.jd.com?reason=403` 或同类拦截 URL / 异常状态码 |
| 滑块验证 | 页面出现滑块验证 DOM 容器 |
| 图形验证码 | 验证码弹层 / 图形验证 DOM 出现 |

## 退避决策树（命中信号后）

```
命中风控信号
  → 立即停（绝不硬重试，硬刚加剧封禁）
  → 暂停一段时间
  → 降频 / 换品类（减少同域高频特征）
  → 仍受阻 → 切平台（京东被封切淘宝，实战验证路径）
  → 滑块 / 验证码 → 升级人工（本 skill 不破解）
```

## 站点差异附录（轻量，逻辑通用）

三站风控判断逻辑本质一致（都在检测"不像真人"），仅入口与签名不同：

| 站点 | 搜索入口 | 关键信号 / 选择器 |
|---|---|---|
| 京东 | `search.jd.com` | 商品卡 `[data-sku]`（非 href）；403 签名 `pc-frequent-pro.pf.jd.com?reason=403`；登录态读用户名节点 |
| 淘宝 | `s.taobao.com/search?q=` | 商品详情 `item.taobao.com`；冷跳详情 URL 是高风控信号 |
| 天猫 | 经淘宝/天猫搜索进入 | 商品详情 `detail.tmall.com` |
